# Security Policy

This repository contains Markdown knowledge materials for **Stunspot's Guide to Semantics, Semiotics, and Symbols**, not executable software.

## Report Sensitive Issues

Please report sensitive issues responsibly if you find:

- Exposed secrets, credentials, private paths, or private contact information.
- Malicious links or suspicious external references.
- Prompt-injection-style text that appears intentionally hostile to AI/RAG ingestion.
- Poisoned corpus content, unsafe instructions, or misleading source artifacts.
- Licensing, attribution, or provenance concerns that should not be handled in a public thread.

For ordinary typos, broken links, citation notes, or clarity improvements, use GitHub Issues instead.

## Scope

In scope:

- Repository files.
- Published Markdown corpus materials.
- GitHub Pages navigation files.
- Manifest, citation, and metadata files.

Out of scope:

- Vulnerabilities in GitHub itself.
- Vulnerabilities in third-party model providers or RAG tools.
- General macroeconomic, legal, financial, or investment advice disputes.
- Runtime bugs in software not contained in this repository.

## Handling Expectations

Security and sensitive corpus-hygiene reports should include the affected file path, relevant excerpt, why it matters, and whether the issue is public-safe or should remain private until reviewed.
