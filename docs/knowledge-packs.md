# Knowledge Packs — Stunspot's Guide to Semantics, Semiotics, and Symbols

The repository separates public navigation from model-ingestion material.

- `docs/` contains GitHub Pages navigation and usage guidance.
- `knowledge-packs/by-report/` contains the 14 canonical source reports.
- `knowledge-packs/compiled-packs/` contains the 4 recommended upload packs.
- `knowledge-packs/omnibus/` contains the whole canon as a single file.

## Recommended Default

Use the **compiled packs** unless you have a specific reason not to. They preserve the canon’s four-volume structure while keeping the upload set manageable.

## Pack Types

| Pack | Files | Best Use | Use When |
|---|---:|---|---|
| Source reports | 14 | Precise retrieval, selective upload, citation, editing, source inspection. | You need granular provenance, report-level retrieval, or targeted inclusion/exclusion. |
| Compiled packs | 4 | Recommended default: grouped coverage with lower file count. | You are loading an AI Project, custom assistant, NotebookLM-style workspace, or RAG corpus with moderate file limits. |
| Omnibus | 1 | One-file import, local archive, whole-corpus search, or strong long-context/RAG systems. | Your system handles large single files well and preserves heading structure during chunking. |

## Volume-to-Pack Map

| Compiled Pack | Source Reports Included | Conceptual Role |
|---|---|---|
| Volume 1. A-E — Foundations of Representation, Meaning, and Evaluation | A, B, C, D, E | Core SSI primitives: representation, signs, linguistic structure, concepts, grounding, evidence, warrant. |
| Volume 2. F-J — Operating Domains of Meaning Construction | F, G, H, I, J | Contextual and cultural meaning: pragmatics, metaphor, discourse, narrative, social reality, myth, ritual, memes, multimodal signs. |
| Volume 3. K-L — Constraint, Instability, and Translation Layers | K, L | Ambiguity, polysemy, semantic drift, translation, domain transfer, and cross-cultural variance. |
| Volume 4. M-N — Failure Diagnosis, Manipulation, and Implementation | M, N | Semantic pathology, manipulation, interpretive failure, knowledge infrastructure, metadata, provenance, routing, and assistant governance. |

## Loading Notes

- Preserve filenames and paths when possible; they encode report order and provenance.
- If your system supports metadata, store report code, title, volume, source path, and pack type.
- Prefer heading-aware chunking. The reports are structured so headings carry semantic scope.
- Use source reports for citation-sensitive work.
- Use compiled packs for general assistant improvement.
- Use the omnibus for archival, local search, or high-capacity contexts.
